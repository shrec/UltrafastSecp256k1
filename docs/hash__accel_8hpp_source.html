5// ============================================================================
6// Accelerated Hashing — SHA-256 / RIPEMD-160 / Hash160
7// ============================================================================
8//
9// ## Three tiers of acceleration (runtime-detected):
10//
11// Tier 0: SCALAR — Portable C++ (baseline, always available)
12// Tier 1: SHA-NI — Intel SHA Extensions (single-message HW accel, ~3-5×)
13// Tier 2: AVX2 — 4-way multi-buffer SHA-256 (interleaved, ~8-12×)
14// + optimized RIPEMD-160 with BMI/BMI2
15// Tier 3: AVX-512 — 8-way multi-buffer SHA-256 (if available, ~16×)
16//
17// ## Hot-path API for search pipeline:
18//
19// Compressed pubkey (33 bytes) → SHA-256 → RIPEMD-160 = Hash160 (20 bytes)
20//
21// - hash160_single(): single pubkey → 20 bytes
22// - hash160_batch(): N pubkeys → N×20 bytes (multi-buffer SIMD)
23// - sha256_33(): SHA-256 of exactly 33 bytes (pubkey-optimized)
24// - ripemd160_32(): RIPEMD-160 of exactly 32 bytes (SHA output)
25//
26// ## Pubkey-specific optimization:
27//
28// SHA-256(33 bytes) always produces exactly 1 block (33 + 1 + 22 + 8 = 64).
29// The padding is constant and can be precomputed. This eliminates all
30// branching, length encoding, and buffer management from the hot path.
31//
32// ============================================================================
