1/* ============================================================================
2 * UltrafastSecp256k1 -- Stable C ABI
3 * ============================================================================
4 *
5 * SINGLE HEADER that exposes the entire public C API.
6 * Include only this file from application / binding code.
7 *
8 * ## Design principles
9 *
10 * 1. Opaque context (`ufsecp_ctx*`) -- all state lives here.
11 * 2. Every function returns `ufsecp_error_t` (0 = OK).
12 * 3. No internal types leak -- all I/O is `uint8_t[]` with documented sizes.
13 * 4. ABI version checked at link time via `ufsecp_abi_version()`.
14 * 5. Thread safety: each ctx is single-thread; create one per thread or
15 * protect externally.
16 * 6. Dual-layer constant-time: secret-dependent operations (scalar mul,
17 * nonce gen, key tweak) ALWAYS use the CT layer; public operations
18 * (verification, point serialisation) ALWAYS use the fast layer.
19 * Both layers are architecturally wired -- no flag, no opt-in.
20 * This eliminates the human factor entirely.
21 *
22 * ## Naming
23 *
24 * ufsecp_<noun>_<verb>() e.g. ufsecp_ecdsa_sign()
25 * UFSECP_<CONSTANT> e.g. UFSECP_PUBKEY_COMPRESSED_LEN
26 *
27 * ## Memory
28 *
29 * Caller always owns output buffers.
30 * Library never allocates on behalf of caller (except ctx create/clone).
31 *
32 * ============================================================================ */

65/* ===========================================================================
66 * Context
67 * ===========================================================================
68 *
69 * Constant-time safety is architectural, NOT flag-based.
70 *
71 * +-------------------------------------------------------------+
72 * | Layer 1 -- FAST: public operations (verify, point arith) |
73 * | Layer 2 -- CT : secret operations (sign, nonce, tweak) |
74 * | Both layers are ALWAYS ACTIVE simultaneously. |
75 * | No opt-in / opt-out. Human-error-proof by design. |
76 * +-------------------------------------------------------------+
77 *
78 * CT layer guarantees:
79 * - Complete addition formula (branchless, 12M+2S)
80 * - Fixed-trace scalar multiplication (no early exit)
81 * - CT table lookup (scans all entries)
82 * - Valgrind/MSAN verifiable (SECP256K1_CLASSIFY / DECLASSIFY)
83 *
84 * =========================================================================== */
