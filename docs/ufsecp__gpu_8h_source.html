1/* ============================================================================
2 * UltrafastSecp256k1 -- GPU Acceleration C ABI
3 * ============================================================================
4 *
5 * Backend-neutral C ABI for GPU-accelerated batch secp256k1 operations.
6 *
7 * ## Design principles
8 *
9 * 1. Opaque GPU context (`ufsecp_gpu_ctx*`) -- backend, device, queue state.
10 * 2. Every function returns `ufsecp_error_t` (0 = OK).
11 * 3. Backend-neutral: CUDA / OpenCL / Metal are implementation details.
12 * 4. No internal GPU types leak -- all I/O is `uint8_t[]` with fixed strides.
13 * 5. Thread safety: each gpu_ctx is single-thread. Create one per thread or
14 * protect externally.
15 * 6. All first-wave operations are PUBLIC-DATA ONLY (verification, hashing,
16 * generator mul). ECDH is secret-bearing and documented as such.
17 *
18 * ## Feature maturity
19 *
20 * This header defines the first-wave GPU API surface. Backend support
21 * per-operation varies:
22 *
23 * CUDA -- all 6 first-wave ops implemented
24 * OpenCL -- 4/6 ops (generator_mul, ecdh, hash160, msm); ECDSA/Schnorr
25 * verify return UNSUPPORTED (needs extended kernel compilation)
26 * Metal -- device discovery / lifecycle only; all ops return UNSUPPORTED
27 *
28 * Operations that a backend does not implement return
29 * UFSECP_ERR_GPU_UNSUPPORTED (104). Callers MUST handle this gracefully.
30 * Backend coverage will expand over subsequent releases.
31 *
32 * Guarantees:
33 * - Discovery + lifecycle functions work on all compiled backends.
34 * - Per-item results for batch ops are well-defined even on partial failure.
35 * - ECDH is the only secret-bearing GPU operation. All others are public-data.
36 * - ABI layout (function signatures, strides, error codes) is stable.
37 * - Backend additions do not break existing calling code.
38 *
39 * ## Memory
40 *
41 * Caller owns all input/output buffers. Library manages device memory
42 * internally and copies results back on return.
43 *
44 * ## Batch layout
45 *
46 * All batch inputs/outputs use flat contiguous arrays with fixed per-item
47 * strides documented in each function.
48 *
49 * ============================================================================ */
